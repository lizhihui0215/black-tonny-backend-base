# Contributing

This file defines the repository-level contribution and collaboration flow for `black-tonny-backend-base`.

- Start new work from a clean branch or clean worktree by default.
- Use the default flow: plan -> implement -> review -> submit PR -> follow up until merge.
- Follow-up work on an existing PR is not a new task, but it must stay inside the current PR goal unless a scope change is explicitly approved or a new PR is requested.
- Do not commit, push, or create a PR until submission is explicitly approved.
- Keep output minimal and prefer repo-local scripts for mechanical artifacts such as diffs, changed-file lists, PR drafts, and migration anchors.
- Align PR descriptions to the current repository template:
  - `PR Goal`
  - `Changed Files`
  - `Boundary Check`
  - `Not Migrated`
  - `Risks`
  - `Validation`
